# 深渊之塔 (Tower of the Abyss)

一款克苏鲁主题的卡牌 Roguelike 游戏，灵感来自《杀戮尖塔》。
核心特色是**理智 / 觉醒系统**：失去理智会觉醒，获得强大但危险的力量。

## 怎么运行

直接在浏览器打开 `index.html` 即可游玩，无需任何安装。
想发布到网上、得到固定链接，见 **[DEPLOY.md](DEPLOY.md)**。

## 项目结构（分层架构）

游戏按职责拆分成多个模块，逻辑与界面解耦，方便长期维护和扩展。

```
tower-of-abyss/
├── index.html       入口：HTML 结构 + 按顺序加载脚本
├── styles.css       全部样式
├── js/
│   ├── data.js      纯数据：卡牌、角色、敌人、事件、觉醒/清明配置
│   ├── game.js      游戏流程与渲染：地图、战斗、奖励、事件
│   ├── save.js      存档系统（localStorage 读写、角色重关联）
│   ├── audio.js     音效系统（基于 Howler，无音频文件时静音降级）
│   └── main.js      启动引导 + 存档/音效整合
└── assets/
    └── audio/       音频文件（可选，见该目录 README）
```

### 各模块职责

- **data.js** — 游戏的全部「内容」。想加新卡牌、新敌人、新事件、新角色，改这里。
- **game.js** — 游戏的「规则与表现」。战斗怎么打、伤害怎么算、界面怎么画。
- **save.js** — 把游戏状态 `G` 存进浏览器、读出来。回到地图时自动存档，死亡/通关时清档。
- **audio.js** — 播放音效和背景音乐。设计为可有可无：没有 Howler 或音频文件时自动禁用，不影响游戏。
- **main.js** — 把上面几块粘起来，处理「继续游戏」「静音按钮」等入口逻辑。

### 加载顺序很重要

`index.html` 里脚本按 `data → game → save → audio → main` 顺序加载。
它们共享同一全局作用域和游戏状态对象 `G`。改动加载顺序可能导致报错。

## 核心系统速览

- **双资源**：生命值 (HP) + 理智值 (SAN)
- **觉醒**：SAN 归零后进入觉醒，卡牌变异、威力大增，但每回合流血、可能失控、敌人变强
- **沉沦**：觉醒后继续「消耗理智」会累积沉沦值，攒满推进到更深的觉醒等级
- **清明**：保持高理智获得护盾、减费、回血等被动收益（觉醒流的对立路线）
- **三角色**：调查员（理智流）、退伍兵（力量/坦克）、先知（觉醒流），各有专属牌组和被动

## 平衡

数值经过模拟器验证（见 `balance_sim.js`，在上一级目录）。
每次改动数值后，可重新跑模拟器确认两条路线胜率是否仍然均衡。
